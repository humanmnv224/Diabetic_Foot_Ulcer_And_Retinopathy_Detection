# Contributing to Diabetic Foot Ulcer and Retinopathy Detection

Thank you for your interest in contributing! This document provides guidelines and instructions.

## Code of Conduct

- Be respectful and professional
- Focus on the research and medical accuracy
- Report issues responsibly

## Getting Started

1. Fork the repository
2. Clone your fork:
   ```bash
   git clone https://github.com/YOUR-USERNAME/Diabetic_Foot_Ulcer_And_Retinopathy_Detection.git
   cd Diabetic_Foot_Ulcer_And_Retinopathy_Detection
   ```

3. Create a virtual environment and install dependencies:
   ```bash
   python -m venv .venv
   source .venv/bin/activate  # On Windows: .\.venv\Scripts\Activate.ps1
   pip install -r requirements.txt
   ```

4. Create a feature branch:
   ```bash
   git checkout -b feature/your-feature-name
   ```

## Development Guidelines

- Keep commits focused and descriptive
- Update README.md if adding new features
- Test locally before pushing
- Follow PEP 8 style guidelines

## Types of Contributions

### Bug Reports
- Use GitHub Issues with a clear title and description
- Include steps to reproduce
- Specify your environment (OS, Python version, CUDA/CPU)

### Feature Requests
- Describe the use case
- Explain expected behavior
- Link to relevant research papers if applicable

### Model Improvements
- Document changes to model architecture
- Report training metrics (loss, accuracy, kappa)
- Include comparative analysis with baseline

### Documentation
- Fix typos and clarify instructions
- Add examples for new features
- Improve troubleshooting section

## Pull Request Process

1. Ensure code passes basic checks
2. Update documentation as needed
3. Reference related issues
4. Wait for maintainer review

## Medical Disclaimer

This project is for research and educational purposes only. Any modifications must maintain the disclaimer that this software is not for clinical diagnosis.
